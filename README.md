# Trimestre-VII
## Monserrath Jiménez
## UEA VII Producción y Comunicación en la Cultura Colaborativa
## Licenciatura en Arte y Comunicación digitales

### El objetivo de este repositorio es reunir y documentar todos los ejercicios técnico-prácticos desarrollados durante la UEA VII "Producción y Comunicación en la Cultura Colaborativa". Su propósito es servir como un espacio centralizado donde se almacenen, organicen y consulten los distintos materiales creados a lo largo del trimestre, facilitando el aprendizaje y el seguimiento de cada práctica. 

### Dentro de este repositorio se encuentran diversos documentos y ejemplos diseñados para apoyar el proceso de aprendizaje en el uso del lenguaje JavaScript, así como otros lenguajes clave para el desarrollo web, como HTML, CSS y JSON. Actualmente, los ejercicios se enfocan especialmente en establecer la conexión entre nuestro proyecto y una base de datos mediante Supabase. Esta integración permite implementar múltiples funcionalidades según las necesidades de cada proyecto del eje trimestral, tales como permitir que diferentes usuarios suban imágenes, textos, audios o videos.

### Cada ejemplo incluye su propio HTML para estructurar la página web, código CSS para definnir estilos y personalizar la interfaz, así como scripts en JavaScript que permiten añadir funciones interactivas o dinámicas. En algunos casos, también se emplean archivos JSON con el fin de facilitar el intercambio de datos dentro del flujo de trabajo. 

### Este repositorio se organiza mediante una estructura de carpetas diseñada para facilitar la búsqueda, consulta y comprensión de cada ejercicio realizado a lo largo del trimestre. Cada carpeta está nombrada ya sea con título del ejercicio o, en algunos casos, con la fecha en que fue elaborado. Esta forma de clasificación permite identificar rápidamente el contenido y tener una idea previa del tipo de práctica o del tema abordado en cada actividad. 

### Dentro de cada crpeta se incluyen los archivos fundamentales para el desarrollo de los ejercicios: un documento HTML que contiene la estructura base de la página, un archivo CSS encargado del diseño visual, y un archivo JavaScript que integra las funciona interactivas necesarias. En algunos ejercicios también se incorpora un archivo JSON.

### Esta organización no solo mantiene el repositorio ordenado, sino que también ofrece una ventaja importante: permite localizar los elementos necesarios para construir o complementar el proyecto eje trimestral, el cual se encuentra actualmente en desarrollo. Gracias a esta distribución clara y consistente, quien consulte el repositorio podrá navegar entre los ejercicios, enetender su propósito y reutilizar partes del código según las necesidades específicas de su proyecto.

### Como el trimestre 25-O para la UEA VII aún está en curso, es posible que el repositorio todavía se perciba un poco confuso o con una organización que podría ser más precisa. Sin embargo, esta estructura, me ha resultado útil para identificar y recordar ejercicios de clases anteriores. Gracias a ello, puedo retomar fragmentos de código y reutilizarlos tanto en el proyecto del eje trimestral como en nuevas actividades que surgen durante las sesiones. Esto ha sido especialmente valioso porque muchas veces permite ahorrar tiempo al adaptar funciones o estilos en lugar de volver a escribirlos desde cero. 

### Conforme avance el trimestre y se vayan agregando más carpetas con distintos ejercicios, será más evidente el proceso de aprendizaje y el conocimeinto acumulado. Visualizar ese progreso también ayuda a reforzar la comprensión de cada tema y a notar las mejoras en la manera de organizar y documentar el trabajo.

### Un aspecto que sigo trabajando es la constancia al subir los cambios. A veces olvido hacer push con regularidad, y sé que esto puede generar problemas si en algún momento intento subir demasiados archivos a la vez. Afortunadamente, por ahora el repositorio no contiene elementos pesados como videos, lo cual evita complicaciones al momento de gestionar los cambios. Aun así, mantener una disciplina en el uso de Git es una habilidad que quiero seguir fortaleciendo.
